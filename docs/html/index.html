NEED AN OVERVIEW Graph Builder 2.0 (alpha) is a library of Apache Pig UDFs (user defined functions) and macros to enable scalable graph ETL in the popular Pig Latin scripting language. GB lets users create graphs from structured, semi-structured and unstructured data. The library offloads many of the complexities of graph construction, such as creating property graph elements from raw data, deduplicate property graph elements, bulk loading a graph to a graph database, and serializing property graph elements in different formats such as edge and vertex list formats, and the RDF format.

Prerequisites

GB 2.0 (alpha) requires Apache Hadoop and Apache Pig installed. To use the LOAD_TITAN macro for bulk loading graphs to Titan, HBase and Titan graph database are also required. In particular, GB 2.0 (alpha) has been tested against Java 1.7, Hadoop 1.2.1, Pig 0.12, Hbase 0.94.12, and Titan 0.4.0.

Building The Source Code

GB 2.0 (alpha) uses Apache Maven as the build manager. You can build GB with

mvn clean package

and graphbuilder-2.0-alpha-with-deps.jar will be created under the target directory. In order to skip unit tests during the build

mvn clean package -DskipTests

Using GB 2.0 (alpha)

To use GB 2.0 (alpha) UDFs, in your Pig script REGISTER graphbuilder-2.0-alpha-with-deps.jar and IMPORT graphbuilder.pig script. The graphbuilder.pig script defines the ExtractJSONField and CreateRowKey UDFs (the UDFs that take no constructor arguments), and LOAD_TITAN and MERGEDUPLICATEGRAPHELEMENTS macros. The user has to define the other UDFs with the constructor arguments they want; please see examples under the examples/ directory for the use of GB functionality in your Pig scripts. Assuming GB 2.0 (alpha) is installed at /usr/local/gb/ add the following to the beginning of your Pig script:

REGISTER /usr/local/gb/target/graphbuilder-2.0-alpha-with-deps.jar;
IMPORT '/usr/local/gb/pig/graphbuilder.pig';

Known Issues:

Using DUMP to print property graph elements fails
Using Pig's DUMP command to dump property graph elements causes "ERROR org.apache.pig.tools.grunt.Grunt - ERROR 2998: Unhandled internal error. java.io.IOException: Could not find class com.intel.hadoop.graphbuilder.graphelements.SerializedGraphElementStringTypeVids, while attempting to de-serialize it". To fix the problem, in $HADOOP_HOME/conf/hadoop-env.sh file update the line that exports HADOOP_CLASSPATH and add "$PIG_HOME/pig-0.12.0.jar:$GB_HOME/target/graphbuilder-2.0-alpha-with-deps.jar" to the end of this classpath definition ($PIG_HOME and $GB_HOME are the paths where Pig and GB are installed).
